STEP 1: Google Fonts
--------------------
Go to: Site Settings > Custom Code > Header Tracking Code
Paste this BEFORE anything else:

<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=DM+Serif+Display:ital@0;1&family=Manrope:wght@300;400;500;600;700;800&display=swap" rel="stylesheet">


STEP 2: Master CSS
------------------
OPTION A -- External Hosting (RECOMMENDED)
  Upload 01-MASTER-CSS.css to a CDN, public GitHub repo, or any
  publicly accessible URL. Then add this to SITE-WIDE Header
  Tracking Code (not per-page):

  <link rel="stylesheet" href="https://YOUR-CDN-URL/01-MASTER-CSS.css">

  WHY: One update propagates to ALL pages instantly. No need to
  paste CSS on every page individually.

OPTION B -- Per-Page (Fallback)
  Go to: Page Settings > Custom CSS
  Paste the ENTIRE contents of: 01-MASTER-CSS.css
  WARNING: You must repeat this on EVERY page. Any CSS change
  requires updating all 6 pages individually.

-------------------------

  Footer Tracking Code:

  Go to: Page Settings > Custom Code > Footer

  </script>

2. GHL PLATFORM NOTES & LIMITATIONS
================================================================

READ THIS BEFORE BUILDING. These are verified GHL behaviors as
of early 2026 that affect how this design system gets implemented.

LIMITATION 1: CSS IS PAGE-LEVEL ONLY
-------------------------------------
GHL does not have a "global CSS" setting. Custom CSS in Page
Settings applies only to that page. This is why external hosting
(Step 2, Option A above) is strongly recommended -- it gives you
site-wide CSS from a single file.

LIMITATION 2: FORMS RENDER IN IFRAMES
--------------------------------------
GHL forms (booking forms, quiz/survey forms, etc.) render inside
iframes. Page-level CSS CANNOT style form fields. To style forms:
  -> Go to the Form Builder for each form
  -> Open Styles -> Custom CSS panel
  -> Apply form-specific styles THERE, not in the page CSS

This means the sp-* classes in the master CSS will NOT affect
form inputs, labels, or buttons inside GHL forms.

IMPORTANT FOR V2: The Getting Started page quiz funnel is a GHL
multi-step survey embedded in an iframe. Style it within the GHL
Survey Builder, not the page CSS.

LIMITATION 3: NO NATIVE SCROLL ANIMATIONS
------------------------------------------
GHL has no built-in animation system. The sp-animate class and
IntersectionObserver JS in 02-MASTER-JS.js is the correct
workaround. Every section that needs fade-in-on-scroll must
either:
  a) Have sp-animate as a Custom Class on a GHL element, OR
  b) Be wrapped in a Custom Code element with sp-animate on it

LIMITATION 4: CUSTOM CLASS FIELD IS INCONSISTENT
-------------------------------------------------
Not all GHL elements expose the "Custom Class" field. For complex
sections (pillar cards, coach features, testimonial cards, trust
bar, science diagram, timeline), use Custom Code elements instead
of relying on native elements with custom classes.

LIMITATION 5: NO STAGING ENVIRONMENT
-------------------------------------
Changes in GHL go live immediately. Before switching DNS to the
new site:
  -> Build on a subdomain (e.g., new.setpointrx.com)
  -> Or use an unpublished page for testing
  -> QA everything before pointing the live domain

GHL NATIVE ELEMENTS -- WHEN TO USE THEM
----------------------------------------
USE native GHL elements for simple sections:
  + Headings (apply sp-heading-xl, sp-heading-lg, etc.)
  + Text blocks (apply sp-body-text, sp-body-large)
  + Buttons (apply sp-btn-primary, sp-btn-secondary)
  + Images (set border-radius in element settings)
  + Google Maps embed (Locations page)
  + Survey/quiz embed (Getting Started page)
  + FAQ accordion (GHL has a native FAQ element -- simpler to
    manage than the custom HTML version. Use native if you want
    easy editing; use custom HTML if you need exact visual control)

USE Custom Code elements for complex layouts:
  x Trust bar with dividers
  x Science section split layout + diagram
  x Pillar cards grid (5 columns)
  x Coach feature section with checkmark list
  x Result/transformation card grid
  x Testimonial card grid
  x Timeline steps (Getting Started)
  x Challenge hero with price + includes grid
  x Stats counters

ELEMENT TEMPLATES (NEW 2025 FEATURE)
-------------------------------------
GHL now lets you save sections as "Element Templates" -- reusable
blocks you can drop into any page. Save these as templates:
  -> Final CTA section (teal background, quiz CTA)
  -> Trust Bar section
  -> Footer section
  -> Mobile Sticky CTA
  -> Guarantee badge

This ensures consistency across all 6 pages and lets you edit
once to update everywhere.
